# Jupyter Ноутбуки для Гибридной Рекомендательной Системы

Этот каталог содержит интерактивные ноутбуки для обучения, тестирования и анализа системы DeepFM+SVD++ + Dueling DQN.

## Архитектура системы

Система состоит из двух основных компонентов:

### 1. Статическая модель: DeepFM+SVD++

Гибридная модель для предсказания многокритериальных оценок:
- **DeepFM компонент**: глубокие факторизационные машины для извлечения высокоуровневых взаимодействий между признаками
- **SVD++ компонент**: факторизация с явным и неявным обратной связью для учета истории взаимодействий пользователя
- **Выход**: предсказание 4 критериев (Mastery, Engagement, SelfRegulation, Outcome)

Математическое представление:
```
y = σ(DeepFM(x) + SVD++(user_history))
где σ - сигмоид, y ∈ [0,1]^4
```

### 2. Динамическая модель: Dueling DQN

Агент обучения с подкреплением для долгосрочного планирования:
- **Архитектура Dueling**: разделение на слой стоимости состояния и слой преимущества действия
- **Action Masking**: маскирование недопустимых действий в зависимости от контекста
- **Prioritized Replay Buffer**: приоритизированное воспроизведение эпизодов с высокой ошибкой

Функция Беллмана:
```
Q(s, a) = r(s, a) + γ * E[V(s') | mask(s')]
где mask предотвращает выбор недопустимых действий
```

### 3. Окружение (среда взаимодействия)

#### Для ITM-Rec (базовый датасет):
- **Состояние**: пользовательское вложение, контекст, история взаимодействий, прогресс
- **Действие**: выбор рекомендуемого учебного материала
- **Вознаграждение**: многокритериальная функция на основе оценок DeepFM
- **Новизна**: бонус за рекомендацию редко встречающихся элементов

#### Для OULAD (расширенный датасет):
- **Состояние**: включает демографию, модуль курса, прогресс недели, прокси-критерии
- **Действие**: выбор между материалами (VLE) и тестами с учетом типа задержки
- **Вознаграждение**: дельта прокси-критериев между последовательными неделями
- **Функция обрыва**: завершение при достижении конца курса или критерия отсева

## Формулы основных метрик OULAD

### Прокси-критерии (многокритериальная аппроксимация):

**Outcome** (интегральный целевой отклик):
```
Outcome = 0.7 * Mastery + 0.3 * final_result_norm
```

**Mastery** (академическое освоение):
```
Mastery = sum(score_i * weight_i) / sum(weight_i)
где score_i - нормализованная оценка за тест i
weight_i - вес теста i в курсе
```

**Engagement** (вовлеченность):
```
Engagement = 0.35 * norm(clicks) 
           + 0.25 * norm(active_days) 
           + 0.20 * norm(unique_resources) 
           + 0.10 * norm(activity_diversity) 
           + 0.10 * norm(regularity)
```

**SelfRegulation** (самоорганизация):
```
SelfRegulation = 0.40 * ontime_ratio 
               + 0.25 * (1 - norm(mean_delay)) 
               + 0.20 * submission_ratio 
               + 0.15 * continuity_ratio
```

где все компоненты нормированы в диапазон [0, 1] через min-max нормализацию.

### Оценочные метрики:

**H1 - Адаптивность к контексту:**
```
AdaptabilityScore = 1 - (σ_P + σ_R) / 2
где σ_P - стандартное отклонение предсказаний в страте
σ_R - стандартное отклонение фактических результатов
```

**H2 - Долгосрочные метрики:**
```
CDR (Cumulative Discounted Reward) = sum(γ^t * r_t)
Retention Rate = (финальные студенты) / (начальные студенты)
Learning Slope = тренд параметра Mastery по времени
```

**H3 - Абляция новизны:**
Сравнение агента: полный vs без контекста vs без демографии vs без новизны.

## Структура ноутбуков

### Быстрый старт

#### **`00_quickstart.ipynb`** - РЕКОМЕНДУЕТСЯ НАЧАТЬ ОТСЮДА

Полный цикл от данных до результатов (10-15 минут):

1. Загрузка конфигурации и данных (ITM-Rec или OULAD)
2. Обучение DeepFM+SVD++
3. Обучение Dueling DQN агента
4. Проверка гипотез H1/H2/H3
5. Визуализация траекторий агента
6. Итоговый отчет с метриками и графиками

**Выход**: полностью обученная система с 18+ графиками и 10+ таблицами метрик.

### Гипотезы и визуализация

#### **`06_hypotheses.ipynb`** - Проверка трех гипотез (H1/H2/H3)

Детальный анализ эффективности системы:

**H1 - Адаптивность к контексту:**
- Расчет AdaptabilityScore для каждого контекстного слоя
- Анализ по стратам: Class, Semester, Lockdown (ITM-Rec) или Phase (OULAD)
- Таблицы стабильности предсказаний
- Тепловая карта адаптивности

**H2 - Долгосрочные метрики:**

- Сравнение агента: полный vs без контекста vs без демографии vs без новизны
- Анализ баланса между разнообразием и релевантностью
- Таблицы влияния каждого компонента

**Рекомендация**: используйте для глубокого анализа эффективности всей системы.

#### **`07_trajectories.ipynb`** - Визуализация поведения агента

Качественный анализ рекомендаций:

- Карты переходов между рекомендациями

## Использование API в ноутбуках

Все ноутбуки строятся на основе унифицированного Python API:

```python
from src import api

# Конфигурация
config = api.build_config("itmrec", yaml_path="configs/itmrec.yaml")
run_dir = api.prepare_run(config, run_name="my_experiment")

# Данные
bundle = api.load_dataset_bundle("itmrec", config=config)

# Обучение
static = api.train_static("itmrec", config=config, bundle=bundle, run_dir=run_dir)
dqn = api.train_dqn("itmrec", config=config, bundle=bundle, 
                    deepfm_model=static["model"], run_dir=run_dir)

# Окружение
env = api.build_environment(bundle, static["model"], config)

# Оценка
results = api.evaluate_system(
    "itmrec", config=config, bundle=bundle,
    deepfm_model=static["model"], dqn_agent=dqn["trainer"].agent,
    run_dir=run_dir
)

# Визуализация
plots = api.visualize_trajectories(trajectories_dict, bundle, run_dir, config)
```
